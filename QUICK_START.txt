  CONFIGURATION WIZARD (configure.sh)

Interactive menu to:
  • Configure MQTT broker (IP, port, credentials)
  • Add devices to track (with MAC address)
  • Set system timeout and preferences
  • Test MQTT connection




From your LOCAL machine:
  bash deploy.sh

It will:
  1. Create deployment package
  2. Stop service on RPi
  3. Backup configuration
  4. Upload new code
  5. Restore config
  6. Restart service
  7. Verify deployment




Flash satellites (one-time):
  1. cd to project directory
  2. source venv/bin/activate
  3. esphome run sat1.yaml --device /dev/ttyACM0
  4. Repeat for sat2, sat3

Assign satellites to rooms:
  http://172.16.9.20/satellites

Check service:
  sudo systemctl status gatekeeper

View logs:
  sudo journalctl -u gatekeeper -f

Test dashboard:
  curl http://172.16.9.20/

Test satellites page:
  curl http://172.16.9.20/satellites




Service won't start:
  sudo journalctl -u gatekeeper -n 50

Port 80 in use:
  sudo netstat -tlnp | grep :80
  sudo kill -9 <PID>

MQTT connection failed:
  Edit: ~/gatekeeper_ng/config/mqtt.json
  Verify broker IP

Bluetooth errors:
  sudo systemctl restart bluetooth
